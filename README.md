# ops-acceptance-validator

这是一个用于“运维验收材料检查”的 Codex skill。

它可以读取用户提供的 Word 和 Excel 验收材料，检查其中是否存在时间范围、数量、子系统名称、厂商表述等不符合要求的内容，并最终只输出可执行的修正建议。

## 适用场景

当用户需要检查以下材料时，使用这个 skill：

- 运维验收材料
- 服务验收文件
- 验收文档
- 运维巡检记录
- 系统清单
- 服务器清单

支持的文件格式：

- `.docx`
- `.xlsx`

## 能检查什么

这个 skill 会重点检查五类问题：

1. 服务周期是否超出范围
2. 子系统数量是否为 19 个
3. 服务器数量是否为 834 台
4. 子系统名称是否符合标准清单
5. 是否出现不应出现的厂商相关表述或具体合作伙伴名称

服务周期固定为：

```text
2025-08-26 至 2026-02-28
```

## 输出要求

最终结果只输出“修正建议”。

有问题时，输出类似：

```markdown
1. “验收材料.docx”中“2026年3月1日”超出服务周期，建议核实实际发生时间并修改为 2026-02-28 及以前的日期。
2. “服务器清单.xlsx”的“巡检记录”工作表中“835台服务器”与标准数量不一致，建议修改为“834台服务器”。
```

没有问题时，只输出：

```markdown
未发现需要修改的内容。
```

不会输出完整检查报告、统计信息、通过项、调试信息或完整修订版文件。

## 使用方式

在 Codex 中，当用户上传或指定 `.docx`、`.xlsx` 验收材料，并要求检查、审核、校验或给出修正建议时，应使用本 skill。

也可以直接运行内置检查脚本：

```bash
python scripts/ops_acceptance_validator.py 验收材料.docx 服务器清单.xlsx
```

脚本会直接在终端输出修正建议。

## 文件读取范围

Word 文件会读取：

- 正文段落
- 表格内容

Excel 文件会读取：

- 所有工作表
- 单元格文本
- 合并单元格中的文字
- 表头、备注、说明性文字

输出建议时，会尽量包含文件名；如果问题来自 Excel，也会尽量包含工作表名称。


- Word 正文和表格读取
- Excel 多工作表读取

- 子系统名称检查
- 厂商泛称检查
- 具体合作伙伴名称替换建议

- 命令行调用和中文输出

## 使用原则

- 只给修正建议，不写检查报告。
- 一条建议只处理一个问题。
- 多个文件按用户提供的顺序输出。
- 同一文件内按原文出现顺序输出。
- 不确定的疑似问题，只建议核实，不强制替换。
- 输出语言面向业务人员，保持简洁、直接、可执行。
